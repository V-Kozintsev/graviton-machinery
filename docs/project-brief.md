# Project Brief

## Цель

Создать самостоятельный демонстрационный сайт-каталог компании GRAVITON, которая продает строительную технику. Сайт должен выглядеть как коммерческий B2B-проект для портфолио.

## Целевая аудитория

Закупщики строительных компаний, руководители участков, механики, представители подрядчиков и дилерских отделов, которым нужно быстро подобрать технику и отправить заявку.

## Пользовательские сценарии

- открыть главную страницу и перейти в каталог;
- отфильтровать технику по категории, производителю, состоянию, наличию и цене;
- открыть детальную страницу машины;
- добавить одну или несколько позиций в заявку;
- заполнить контактные данные и отправить демонстрационную заявку;
- перейти на контакты или вернуться в каталог.

## Страницы

- главная;
- каталог;
- детальная страница техники;
- заявка;
- контакты;
- 404 внутри приложения;
- design board и mockup-экраны для проектирования.

## Функциональность

- адаптивный header и мобильное меню;
- каталог минимум из 12 машин;
- фильтры, сортировка, активные фильтры и пустой результат;
- мобильный drawer фильтров;
- карточки техники со стабильными размерами;
- галерея товара с миниатюрами и клавиатурным управлением;
- localStorage-заявка;
- форма заявки на React Hook Form и Zod;
- toast-уведомления;
- экран успешного демонстрационного завершения.

## Структура Каталога

Категории: экскаваторы, фронтальные погрузчики, бульдозеры, автокраны, дорожная техника, мини-погрузчики. В карточках используются реалистичные демонстрационные характеристики без копирования реальных объявлений.

## Технологии

React, TypeScript, Vite, Tailwind CSS, pnpm, React Router HashRouter, Lucide React, Embla Carousel, React Hook Form, Zod, localStorage, ESLint.

## Ограничения

Нет backend, базы данных, CMS, авторизации, оплаты, CRM и реальной отправки формы. Сайт рассчитан на GitHub Pages и использует hash-маршруты.

## Критерии Готовности

- визуально законченный B2B-каталог;
- сохранены дизайн-система, layout spec и PNG-макеты;
- фильтры, сортировка, детальные страницы и заявка работают;
- изображения адаптивны и оптимизированы;
- интерфейс проверен на 360px;
- `pnpm lint` и `pnpm build` завершаются без ошибок;
- есть GitHub Pages workflow и README.
